  ┌─────────────────────────────────────────────────────────────────────────────────┐
  │                     CIRCUIT BREAKER PATTERN — MICROSERVICE                      │
  └─────────────────────────────────────────────────────────────────────────────────┘

  ┌─────────────┐       HTTP call        ┌──────────────────────────────────────────────┐
  │             │  ────────────────────> │           CIRCUIT BREAKER                    │
  │   Order     │                       │                                              │
  │   Service   │                       │   @CircuitBreaker(name = "inventoryCB",   │
  │  (Caller)   │                       │    fallbackMethod = "fallbackInventory")  │
  │             │  <─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │                                              │
  └─────────────┘   fallback response   └──────────────────────────────────────────────┘
                                                          |
                          ┌───────────────────────────────┼───────────────────────────────┐
                          |                               |                               |
                    [ CLOSED ]                    [ OPEN ]                    [ HALF-OPEN ]
                          |                               |                               |
                  Normal operation            Trips after N% failures       Trial request
                  Calls go through            Returns fallback instantly     sent to check
                  Counts failures             Waits waitDurationInOpen       if service is
                          |                               |                                back up
                          |                               |                               |
                          v                               v                               v
                ┌─────────────────┐       ┌─────────────────┐       ┌─────────────────────┐
                │  Inventory      │       │  Fallback       │       │  Inventory          │
                │  Service        │       │  Method         │       │  Service            │
                │  (Downstream)   │       │  (cached data / │       │  (Downstream)       │
                │                 │       │   default resp) │       │                     │
                └─────────────────┘       └─────────────────┘       └─────────────────────┘
                                                                      success -> CLOSED
                                                                      failure -> OPEN again


  STATE TRANSITIONS:
  ==================
                                failure rate >= threshold
           [ CLOSED ] ─────────────────────────────────────────────> [ OPEN ]
               ^                                                           |
               |   success                                 waitDuration    |
          [ HALF-OPEN ] <─────────────────────────────────────────────── |
               |
               |   failure
           [ OPEN ] <──────────────────────────────────────────────────────

  =================================

        slidingWindowType: COUNT_BASED      # COUNT_BASED or TIME_BASED
        slidingWindowSize: 10              # last 10 calls evaluated
        failureRateThreshold: 50           # >50% failures -> OPEN
        waitDurationInOpenState: 10s       # stay OPEN for 10s
        permittedNumberOfCallsInHalfOpenState: 3  # trial calls
        automaticTransitionFromOpenToHalfOpenEnabled: true
        slowCallDurationThreshold: 2s      # calls > 2s are "slow"
        slowCallRateThreshold: 60          # >60% slow -> OPEN
        recordExceptions:
          - java.io.IOException
          - java.util.concurrent.TimeoutException
          - feign.FeignException

          - com.demo.exception.BusinessException  # don't trip CB on business errors

CircuitBreakerConfig.java — Event Listeners
@Configuration
public class CircuitBreakerEventConfig {

    private static final Logger log =
        LoggerFactory.getLogger(CircuitBreakerEventConfig.class);

    /**
     * Registers state-change listeners on the CB registry.
     * Fires when CB transitions: CLOSED -> OPEN -> HALF_OPEN -> CLOSED
     */
    @Bean
    public CircuitBreakerRegistry circuitBreakerRegistry() {
        CircuitBreakerRegistry registry = CircuitBreakerRegistry.ofDefaults();

        registry.getEventPublisher()
            .onEntryAdded(entry -> {
                CircuitBreaker cb = entry.getAddedEntry();

                // Log every state transition
                cb.getEventPublisher()
                    .onStateTransition(event ->
                        log.warn("CircuitBreaker [{}] state changed: {} -> {}",
                            event.getCircuitBreakerName(),
                            event.getStateTransition().getFromState(),
                            event.getStateTransition().getToState()))

                    .onFailureRateExceeded(event ->
                        log.error("CircuitBreaker [{}] failure rate: {}%",
                            event.getCircuitBreakerName(),
                            event.getFailureRate()))

                    .onCallNotPermitted(event ->
                        log.warn("CircuitBreaker [{}] is OPEN — call rejected",
                            event.getCircuitBreakerName()));
            });

        return registry;
    }
}
OrderService.java — @CircuitBreaker with Fallback
@Service
public class OrderService {

    private static final Logger log =
        LoggerFactory.getLogger(OrderService.class);

    @Autowired
    private InventoryClient inventoryClient;   // Feign client

    @Autowired
    private OrderRepository orderRepository;


    // ── 1. Circuit Breaker wraps the Feign call ──────────────────
    @CircuitBreaker(name = "inventoryCB", fallbackMethod = "fallbackInventory")
    public InventoryResponse checkInventory(String productId) {
        log.info("Calling Inventory Service for productId={}", productId);
        return inventoryClient.getStock(productId);  // may throw FeignException
    }


    // ── 2. Fallback — same signature + Throwable as last param ────
    public InventoryResponse fallbackInventory(String productId, Throwable t) {
        log.warn("Fallback triggered for productId={}, reason={}",
            productId, t.getMessage());

        // Return cached/default data instead of propagating the error
        return InventoryResponse.builder()
            .productId(productId)
            .available(false)
            .quantity(0)
            .message("Inventory service unavailable. Please retry shortly.")
            .build();
    }


    // ── 3. Place order — uses CB-protected inventory check ────────
    public OrderResponse placeOrder(OrderRequest request) {
        InventoryResponse inventory = checkInventory(request.getProductId());

        if (!inventory.isAvailable()) {
            throw new BusinessException("Product out of stock: " + request.getProductId());
            // BusinessException is in ignoreExceptions — won't trip the CB
        }

        Order order = Order.builder()
            .orderId(UUID.randomUUID().toString())
            .productId(request.getProductId())
            .quantity(request.getQuantity())
            .status("PLACED")
            .build();

        orderRepository.save(order);

        return OrderResponse.builder()
            .orderId(order.getOrderId())
            .status(order.getStatus())
            .inventoryMessage(inventory.getMessage())
            .build();
    }
}

OrderController.java
@RestController
@RequestMapping("/orders")
public class OrderController {

    @Autowired
    private OrderService orderService;


    // POST /orders — place a new order
    @PostMapping
    public ResponseEntity<OrderResponse> placeOrder(
            @RequestBody OrderRequest request) {

        OrderResponse response = orderService.placeOrder(request);
        return ResponseEntity.ok(response);
    }


    // GET /orders/inventory/{productId} — check stock directly
    @GetMapping("/inventory/{productId}")
    public ResponseEntity<InventoryResponse> checkInventory(
            @PathVariable String productId) {

        InventoryResponse response = orderService.checkInventory(productId);
        return ResponseEntity.ok(response);
    }


    // Global exception handler for BusinessException
    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<String> handleBusiness(BusinessException ex) {
        return ResponseEntity
            .status(HttpStatus.UNPROCESSABLE_ENTITY)
            .body(ex.getMessage());
    }
}

Concept State What to say in interview
Three States CLOSED OPEN HALF-OPEN CLOSED = normal flow. OPEN = fail-fast, returns fallback immediately. HALF-OPEN = trial period, sends limited calls to test recovery.
Why not try-catch? Design try-catch still makes the failing call every time — hammering a down service. CB stops calling it altogether during OPEN, giving downstream time to recover.
Fallback signature Rule Must match the protected method's return type and parameters, plus one extra Throwable at the end. Resilience4j resolves it by name at startup.
ignoreExceptions Config Business exceptions (stock out, validation) should NOT trip the CB — only infrastructure failures (timeout, connection refused) should. Use ignoreExceptions for this.
slidingWindowType Config COUNT_BASED = last N calls. TIME_BASED = calls in last N seconds. COUNT_BASED is simpler to reason about in interviews.
Slow call threshold Config CB can also open on slow calls, not just failures — crucial for latency-sensitive systems. A slow inventory call can back-pressure your whole order pipeline.
Actuator endpoint Observability /actuator/circuitbreakers shows live state. /actuator/circuitbreakerevents shows full transition history — essential for production debugging.
@CircuitBreaker vs programmatic Design Annotation is cleaner for service methods. Programmatic API (CircuitBreaker.decorateSupplier) is better when you need fine-grained control or reactive pipelines.
